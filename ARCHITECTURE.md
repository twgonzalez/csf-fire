# JOSH Map Architecture: CDN-hosted Rendering + Per-city Data

## Why this architecture exists

At 50+ cities, the original monolithic `demo_map.html` (~9 MB) approach breaks down.
Every visualization update (UI change, bug fix, new engine feature) required rebuilding and
redistributing all city maps. The split architecture separates:

- **Shared rendering code** → CDN-hosted `app.js` (update once, all cities update automatically)
- **City-specific data** → inlined `window.JOSH_DATA` in each city's `demo_map.html`

## Components

### `window.JOSH_DATA` (inlined in `demo_map.html`)

Per-city data inlined as a `<script>` block before the CDN `<script src>` tag. `app.js` reads
this synchronously on parse — no fetch calls, no CORS issues, works from `file://`.

```json
{
  "schema_version": 1,
  "app_js_version": "v1",
  "graph": {
    "built_at": "...",
    "parameters_version": "4.11",
    "nodes": [{"id": 123, "lon": -122.27, "lat": 37.87}, "..."],
    "edges": [{"u": 123, "v": 456, "osmid": "...", "len_m": 85.2, "speed_mph": 25,
               "eff_cap_vph": 394.0, "fhsz_zone": "vhfhsz", "haz_deg": 0.35}, "..."],
    "exit_nodes": [123, 456, "..."]
  },
  "parameters": {
    "parameters_version": "4.11",
    "unit_threshold": 15,
    "mobilization_rate": 0.90,
    "vehicles_per_unit": 2.5,
    "safe_egress_window": {"vhfhsz": 45, "high_fhsz": 90, "moderate_fhsz": 120, "non_fhsz": 120},
    "max_project_share": 0.05,
    "hazard_degradation": {"vhfhsz": 0.35, "high_fhsz": 0.5, "moderate_fhsz": 0.75, "non_fhsz": 1.0},
    "egress_penalty": {"threshold_stories": 4, "minutes_per_story": 1.5, "max_minutes": 12}
  },
  "fhsz": {"type": "FeatureCollection", "features": ["...CAL FIRE FHSZ GeoJSON..."]},
  "briefs": {}
  // v4.11: briefs dict is empty. Briefs are generated client-side
  // by BriefRenderer.render() in sidebar.js on demand.
}
```

**`schema_version`** — integer. `app.js` emits `console.warn` on mismatch so stale maps are
detectable without silent failures. Start at 1.

**`app_js_version`** — string matching the CDN URL path segment (`"v1"`). Informational only;
used for debugging. Set from `_APP_JS_VERSION` in `agents/export.py`.

### `static/v1/app.js` (shared rendering bundle)

CDN URL: `https://twgonzalez.github.io/josh/static/v1/app.js`

Generated by `agents/export.py:export_app_js()` during the `demo` command. Sections (in order):

| Section | Source | Purpose |
|---------|--------|---------|
| Schema compatibility check | `export_app_js()` | `console.warn` if schema_version mismatch |
| WhatIfEngine IIFE | `static/whatif_engine.js` (verbatim) | Dijkstra + ΔT algorithm |
| What-if UI panel injector | `_build_whatif_ui_html()` in `demo.py` | DOMContentLoaded DOM append |
| What-if UI controller | `_build_whatif_ui_js()` in `demo.py` | Pin drop, drag, evaluate, render |
| Brief modal overlay injector | `_build_brief_modal_overlay_js()` in `export.py` | Static modal chrome |
| Brief modal controller | `_build_brief_modal_controller_js()` in `export.py` | show(html) from sidebar.js or legacy filename lookup |

`app.js` is committed to the repo at `static/v1/app.js` and deployed to GitHub Pages via
the CI workflow (`cp -r static output/static`).

### Delivery: inline vs. CDN

`_inject_josh_data_bundle()` uses a **build-time check** to decide how to load `app.js`:

```
static/v1/app.js on disk?
  YES → inline it into demo_map.html    — fully self-contained, works file:// offline
  NO  → <script src="{CDN_URL}" defer>  — loads from GitHub Pages, requires internet
```

In a normal local build `export_app_js()` runs first so the file always exists → inlined.
The CDN fallback activates if an HTML is distributed without the `static/` directory alongside
it, or in a future scenario where CDN-only delivery is preferred (50-city auto-updates).

### `static/whatif_engine.js` (intermediate generated file)

Generated by `export_whatif_engine_js()` during `analyze`. Contains the WhatIfEngine IIFE only.
Used directly by the anti-divergence test suite (`tests/test_whatif_engine.js`). Embedded
verbatim into `app.js` by `export_app_js()`.

**Never edit `static/whatif_engine.js` or `static/v1/app.js` directly** — they are overwritten
on every `analyze`/`demo` run.

## CDN Versioning Scheme

```
https://twgonzalez.github.io/josh/static/v1/app.js
                                              ^^
                                     major version in path
```

### Within a major version (backward-compatible updates)

Deploy `app.js` to `static/v1/app.js` in-place. All cities loading `v1` get the update
automatically on next open. No HTML rebuild needed.

**Examples:**
- Bug fix in what-if panel results display
- New field in the results table
- Brief modal styling change
- WhatIfEngine algorithm fix that doesn't change `JOSH_DATA` schema

### Bumping the major version (backward-incompatible changes)

When `JOSH_DATA` schema changes in a way that a `v1` `app.js` cannot consume a `v2` HTML:

1. Increment `_APP_JS_VERSION = "v2"` in `agents/export.py`
2. Update `_APP_JS_CDN_URL` in `agents/visualization/demo.py` to `.../v2/app.js`
3. Run `demo` for all cities → generates `static/v2/app.js` + new HTMLs referencing `v2`
4. Deploy — `v1` HTMLs keep loading `v1/app.js` indefinitely (old path still exists)

### What NOT to put in `JOSH_DATA` (keep schema stable)

Fields that can be computed at runtime from existing fields do not need to be added to the
schema. The schema is stable when fields are only added (never removed/renamed within a
major version).

## Build Pipeline

```
analyze --city X
  ├─ Download roads, FHSZ, boundary → data/{city}/
  ├─ HCM capacity analysis → roads.gpkg with eff_cap_vph
  ├─ export_graph_json()     → output/{city}/graph.json
  ├─ export_parameters_json() → output/{city}/parameters.json
  ├─ export_fhsz_json()      → output/{city}/fhsz.json
  └─ export_whatif_engine_js() → static/whatif_engine.js  ← shared, not per-city

demo --city X
  ├─ evaluate_project() for each project in {city}_demo.yaml (pre-bakes JOSH_DATA.projects)
  ├─ export_test_vectors()   → output/{city}/test_vectors.json
  ├─ create_demo_map() [Folium] → output/{city}/demo_map.html  (base map)
  ├─ export_app_js()
  │    ├─ export_whatif_engine_js() → static/whatif_engine.js (refreshed)
  │    └─ concat sections    → static/v1/app.js  ← shared, committed to repo
  └─ _inject_josh_data_bundle()
       ├─ reads graph.json, parameters.json, fhsz GeoJSON
       └─ inlines window.JOSH_DATA + app.js → demo_map.html
  # v4.11: briefs and audit trails are generated client-side (sidebar.js + BriefRenderer)
```

## What Requires a Full Python Rebuild

After this architecture change, these still require `demo --city X`:

| Component | Reason |
|-----------|--------|
| Road network GeoJSON (8+ MB) | Folium-baked during map generation |
| Popup HTML per road segment | Per-feature GeoJSON properties, Python-generated |
| Project markers + AntPath traces | Per-project Folium layers |
| Demo panel (left sidebar) | `_build_demo_panel_html()` — data-dependent |
| Brand header + legend | Python-generated (minor; could move to `app.js`) |
| Brief HTML content | Inlined into `window.JOSH_DATA.briefs` by Python |
| City graph/parameters data | `window.JOSH_DATA.graph`, `.parameters`, `.fhsz` |
| `what_happened` overlay | City-specific; `_inject_what_happened_layer()` |

**Future work:** Moving popup HTML generation to client-side JS (reading from GeoJSON feature
properties) would eliminate the rebuild requirement for popup changes — the largest remaining
class of visualization updates that require per-city rebuilds.

## Offline Use

When `static/v1/app.js` is present at build time (all normal local builds), `demo_map.html`
is **fully self-contained** — `app.js` is inlined and the map works from `file://` with no
internet required (beyond the CartoDB tile background and Leaflet, which were always CDN-loaded).

Internet is only needed when `app.js` was not found at build time and the CDN fallback tag was
emitted instead.
